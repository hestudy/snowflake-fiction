# 流水账检测

专门检测小说中的"流水账"问题——只有事件罗列，没有冲突、情绪、期待感。

## 为什么流水账是致命问题？

| 流水账特征 | 读者反应 | 数据影响 |
|------------|----------|----------|
| 无冲突 | 看不下去 | 追读率暴跌 |
| 无情绪 | 无法共情 | 完读率下降 |
| 无期待感 | 看完就忘 | 书架转化低 |
| 信息堆砌 | 直接跳过 | 跳出率飙升 |

## 使用方式

```
/boring-detect [文本内容]
```

## 检测维度

### 1. 无冲突检测

```
❌ "他起床，刷牙，吃早饭，出门上班，坐地铁，到公司打卡..."
   → 纯时间线推进，没有意外，没有阻碍

✅ "他刚刷完牙，手机响了——房东的催租电话。
   他看了一眼余额：382块。房租要2000。"
   → 立刻出现需要解决的问题
```

### 2. 无情绪检测

```
❌ "他走进房间，看到桌上的信，打开看了一遍，然后放下。"
   → 纯动作，不知道他在想什么

✅ "他走进房间，看到桌上的信——手抖了一下，差点拿不住。
   撕开信封的时候，他甚至不敢呼吸。"
   → 情绪层层递进
```

### 3. 无期待感检测

```
❌ "这就是他的一天。第二天，他又重复了同样的生活。"
   → 读者完全不想知道后面

✅ "他不知道的是，那封信里藏着改变他命运的机会。
   而现在，他正准备把它扔进垃圾桶。"
   → 读者：别扔！快捡回来！
```

## 流水账程度分级

| 等级 | 分数 | 描述 | 建议 |
|------|------|------|------|
| 无流水账 | 80-100 | 内容扎实 | 保持即可 |
| 轻微 | 60-79 | 偶有平淡 | 局部优化 |
| 中等 | 40-59 | 多处问题 | 重点改写 |
| 严重 | 0-39 | 通篇流水账 | 需要重构 |

## 流水账修复公式

| 问题 | 修复方案 |
|------|----------|
| 无冲突 → 加意外 | 事件A → 意外/阻碍 → 情绪反应 → 解决尝试 |
| 无情绪 → 加反应 | 他看到了那封信 → 手抖了一下，差点拿不住 |
| 无期待感 → 加悬念 | 这就是他的一天 → 他不知道的是，真正的麻烦才刚刚开始 |
| 信息堆砌 → 加互动 | 大段说明 → 通过对话展示 |
