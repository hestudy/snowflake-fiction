# 流水账检测

专门检测小说中的"流水账"问题——只有事件罗列，没有冲突、情绪、期待感。

## 路由规则

根据用户输入类型，路由到不同处理器：

| 输入类型 | 示例 | 路由目标 |
|---------|------|----------|
| 纯文本 | `/boring-detect [文本内容]` | `skills/boring-detect/SKILL.md`（直接检测） |
| 文件路径 | `/boring-detect ./我的小说/` | `agents/boring-detect.md`（文件处理器） |
| 路径+章节 | `/boring-detect ./我的小说/ 第3-5章` | `agents/boring-detect.md`（文件处理器） |
| 仅章节号 | `/boring-detect 第3章` | `agents/boring-detect.md`（当前目录查找） |

### 判断逻辑

```
输入包含路径（./、/、相对路径）→ 调用 agent
输入包含"第X章"、"chapter"等关键词 → 调用 agent
输入是纯文本 → 调用 skill（直接检测）
```

## 为什么流水账是致命问题？

| 流水账特征 | 读者反应 | 数据影响 |
|------------|----------|----------|
| 无冲突 | 看不下去 | 追读率暴跌 |
| 无情绪 | 无法共情 | 完读率下降 |
| 无期待感 | 看完就忘 | 书架转化低 |
| 信息堆砌 | 直接跳过 | 跳出率飙升 |

## 使用方式

```
/boring-detect [待检测的文本]

# 自然语言触发
"帮我检测这段是不是流水账"
"检查一下流水账"
"这段内容太平淡了"

# 文件模式
/boring-detect ./我的小说/                    # 交互式选择章节
/boring-detect ./我的小说/ 第3章              # 检测指定章节
/boring-detect ./我的小说/ 第3-5章            # 检测章节范围
/boring-detect ./我的小说/ 全部               # 检测所有章节
```

## 流水账程度分级

| 等级 | 分数 | 描述 | 建议 |
|------|------|------|------|
| 无流水账 | 80-100 | 内容扎实 | 保持即可 |
| 轻微 | 60-79 | 偶有平淡 | 局部优化 |
| 中等 | 40-59 | 多处问题 | 重点改写 |
| 严重 | 0-39 | 通篇流水账 | 需要重构 |

## 流水账修复公式

| 问题 | 修复方案 |
|------|----------|
| 无冲突 → 加意外 | 事件A → 意外/阻碍 → 情绪反应 → 解决尝试 |
| 无情绪 → 加反应 | 他看到了那封信 → 手抖了一下，差点拿不住 |
| 无期待感 → 加悬念 | 这就是他的一天 → 他不知道的是，真正的麻烦才刚刚开始 |
| 信息堆砌 → 加互动 | 大段说明 → 通过对话展示 |
