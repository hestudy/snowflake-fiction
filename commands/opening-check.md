# 开篇质量检查

专门检查小说开篇（前三章）的质量，基于"黄金三章"法则。

## 路由规则

根据用户输入类型，路由到不同处理器：

| 输入类型 | 示例 | 路由目标 |
|---------|------|----------|
| 纯文本 | `/opening-check [文本内容]` | `skills/opening-check/SKILL.md`（直接检查） |
| 文件路径 | `/opening-check ./我的小说/` | `agents/opening-check.md`（文件处理器） |
| 路径+章节 | `/opening-check ./我的小说/ 第1-3章` | `agents/opening-check.md`（文件处理器） |
| 仅章节号 | `/opening-check 第1章` | `agents/opening-check.md`（当前目录查找） |

### 判断逻辑

```
输入包含路径（./、/、相对路径）→ 调用 agent
输入包含"第X章"、"chapter"等关键词 → 调用 agent
输入是纯文本 → 调用 skill（直接检查）
```

## 为什么开篇决定生死？

| 数据 | 说明 |
|------|------|
| 读者平均 **3秒** 决定是否弃书 | 第一章前300字是生死线 |
| 前三章流失率 **>80%** | 大部分读者在这里放弃 |
| 第一章有金手指/危机 → 留存率高 **40%** | 开篇设计直接影响推荐 |

## 黄金三章法则

### 第一章：3秒抓眼球（40分）

**核心任务**：从"炸楼"开始写，让读者立刻紧张起来

**开篇公式**：`环境异变 + 主角反常行为 + 致命威胁`

**检查清单**：
- [ ] 前300字有爆点（危机/金手指/反转）
- [ ] 主角已登场
- [ ] 避免寡淡开场（不是"XX是个普通人..."）
- [ ] 避免信息轰炸（不大段倾倒世界观）
- [ ] 避免拖沓入题（不写三段天气描写）

### 第二章：强化冲突（30分）

**核心任务**：给主角一个紧迫目标 + 设置具体阻碍

**结构公式**：`目标可视化 + 阻碍具体化 = 读者替主角着急`

**检查清单**：
- [ ] 有紧迫目标（有时间限制）
- [ ] 有具体阻碍
- [ ] 世界观通过剧情展开
- [ ] 金手指/机遇足够吸引

### 第三章：爽点反馈（30分）

**核心任务**：让读者尝到甜头，即使只是小胜利

**爽点设计公式**：`铺垫（被看不起） → 反转（展示实力） → 反应（围观群众震惊）`

**检查清单**：
- [ ] 有小爽点（打脸小反派/突破小困境/展示隐藏技能）
- [ ] 爽点与前两章呼应
- [ ] 结尾有钩子（悬念/转折/期待/危机）
- [ ] 让读者想看下一章

## 评分标准

| 等级 | 分数 | 预计留存率 | 建议 |
|------|------|------------|------|
| 优秀 | 80-100 | 高 | 保持即可 |
| 良好 | 60-79 | 中 | 针对性优化 |
| 及格 | 40-59 | 中低 | 需要改写 |
| 不及格 | 0-39 | 低 | 建议重写 |

## 使用方式

```
/opening-check [文本内容]

# 文件模式
/opening-check ./我的小说/                    # 自动检查前三章
/opening-check ./我的小说/ 第1章              # 检查指定章节
/opening-check ./我的小说/ 第1-3章            # 检查章节范围
```

## 推荐工作流

```
1. /opening-check [前三章]
2. 根据报告定位问题
3. 逐章修复
4. /opening-check [修改后的内容]  # 再次检查
5. /quality-check [前三章]        # 全面质量检查
```
