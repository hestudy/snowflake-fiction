# 创意与选题检查

检测小说选题和创意相关的常见问题——题材脱节、缺乏辨识度、混搭不当、设定过多。

## 路由规则

根据用户输入类型，路由到不同处理器：

| 输入类型 | 示例 | 路由目标 |
|---------|------|----------|
| 纯文本 | `/concept-check [书名 + 简介 + 内容]` | `skills/concept-check/SKILL.md`（直接检测） |
| 指定检查项 | `/concept-check --辨识度 [书名 + 简介]` | `skills/concept-check/SKILL.md`（指定维度） |
| 优化模式 | `/concept-check --优化 [书名 + 简介]` | `skills/concept-check/SKILL.md`（优化建议） |
| 文件路径 | `/concept-check ./我的小说/` | `agents/concept-check.md`（文件处理器） |
| 路径+章节 | `/concept-check ./我的小说/ 第1-3章` | `agents/concept-check.md`（文件处理器） |
| 仅章节号 | `/concept-check 第1章` | `agents/concept-check.md`（当前目录查找） |

### 判断逻辑

```
输入包含路径（./、/、相对路径）→ 调用 agent
输入包含"第X章"、"chapter"等关键词 → 调用 agent
输入是纯文本/书名+简介 → 调用 skill（直接检测）
输入包含 --优化 → 调用 skill（优化模式）
输入包含 --题材/--辨识度/--混搭/--世界观 → 调用 skill（指定维度）
```

## 为什么选题问题致命？

| 问题 | 读者反应 | 平台影响 |
|------|----------|----------|
| 题材不匹配 | 找不到想看的 | 标签投诉、退订 |
| 无辨识度 | 为什么要看你的？ | 推荐转化低 |
| 混搭不当 | 四不像 | 读者定位混乱 |
| 设定过多 | 记不住 | 弃书率高 |

## 使用方式

```
/concept-check [书名 + 简介 + 前三章]

# 指定检查项
/concept-check --题材 [书名 + 简介 + 内容]
/concept-check --辨识度 [书名 + 简介]
/concept-check --混搭 [书名 + 简介 + 内容]
/concept-check --世界观 [内容]

# 优化模式
/concept-check --优化 [书名 + 简介]

# 自然语言触发
"帮我检查一下选题"
"这个创意有没有辨识度"
"题材和内容匹配吗"

# 文件模式
/concept-check ./我的小说/                    # 交互式选择章节
/concept-check ./我的小说/ 第1-3章            # 检测指定章节
/concept-check ./我的小说/ 全部               # 检测所有章节
```

## 创意评分等级

| 等级 | 分数 | 描述 | 市场表现预测 |
|------|------|------|--------------|
| 优秀 | 80-100 | 创意独特，卖点清晰 | 潜力大 |
| 良好 | 60-79 | 有亮点但可加强 | 表现中等 |
| 及格 | 40-59 | 存在同质化问题 | 需要优化 |
| 不及格 | 0-39 | 缺乏辨识度 | 建议重构 |
