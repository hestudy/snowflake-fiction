# 小说复核

对已创作的小说内容进行质量检查，主要解决长篇小说创作中的**一致性维护**问题。

## 使用方式

```bash
/novel-review                              # 交互式引导
/novel-review ./novel-output/我的小说/      # 指定项目目录
/novel-review 第10章                        # 只检查指定章节
/novel-review 第5-10章                      # 检查章节范围
```

### 指定检查项

```bash
/novel-review --角色        # 只检查角色一致性
/novel-review --时间线      # 只检查时间线
/novel-review --设定        # 只检查设定一致性
/novel-review --大纲        # 只检查大纲偏离
/novel-review --伏笔        # 只检查伏笔回收
/novel-review --文风        # 只检查文风一致性
/novel-review --开篇        # 只检查开篇质量
/novel-review --评估准备    # 推荐评估准备检查（番茄平台）
/novel-review --全文        # 全面检查
```

## 检查项目

| 检查项 | 说明 |
|--------|------|
| 角色一致性 | 性格、对话、能力、关系前后一致 |
| 时间线 | 时间流逝、季节、年龄连贯 |
| 设定一致性 | 能力体系、物品、地点、规则 |
| 大纲偏离 | 核心事件、支线、节奏是否符合规划 |
| 伏笔回收 | 埋设、回收、超期预警 |
| 文风一致性 | 视角、语言风格、AI痕迹、流水账检测 |
| 开篇质量 | 黄金三章检查（仅前3章） |
| 评估准备 | 番茄平台推荐评估准备度检查 |

## 子代理架构

采用子代理架构，将检查任务拆分为独立的小任务，避免上下文溢出：

```
主代理 → 第1批：角色 + 设定 → 第2批：时间线 + 伏笔 → 第3批：大纲偏离 → 第4批：文风 → 汇总报告
```

## 并发控制

```bash
/novel-review --parallel 1    # 串行执行（最保守）
/novel-review --parallel 2    # 每批2个（默认）
/novel-review --parallel 3    # 每批3个（可能限流）
```

| 参数 | 说明 |
|------|------|
| `--parallel N` | 每批并行子代理数，默认 **2** |

**推荐**：日常检查使用单项（如 `--角色`），全面检查使用 `--parallel 1` 串行模式。
