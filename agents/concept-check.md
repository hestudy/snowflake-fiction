# 创意与选题检查文件处理器

自主扫描小说目录、解析章节范围、并行派发子代理逐章检测选题问题。

---

## 第一步：识别输入类型

判断用户输入：

```
输入包含路径（./、/、相对路径）→ 文件模式
输入包含"第X章"、"chapter"等关键词 → 章节定位模式
```

---

## 第二步：发现小说目录结构

使用 `Glob` 工具扫描目录，识别章节文件：

```
优先匹配模式（按顺序尝试）：
1. [路径]/正文/第*章*.md
2. [路径]/chapters/第*章*.md
3. [路径]/第*章*.md
4. [路径]/**/第*章*.md
5. [路径]/chapter-*.md
6. [路径]/chapters/*.md
```

如果找不到章节文件，告知用户未发现章节文件，请确认路径。

同时读取小说的基础信息文件（如有）：
```
[路径]/概念/*.md          # 一句话概括、五句式大纲等
[路径]/concept/*.md
[路径]/大纲/*.md
[路径]/outline/*.md
```

---

## 第三步：解析章节范围

```
"第1章"     → 只检测第1章文件
"第1-3章"   → 检测第1、2、3章文件
"全部"      → 检测所有发现的章节文件
无章节指定  → 列出所有章节，让用户选择
```

---

## 第四步：并行子代理检测

使用 Task 工具派发子代理，每个子代理独立检测一章。

### 并发策略

- 每个章节由一个独立的 Task 子代理处理
- 并发限制：最多 3 个子代理同时运行（避免 API 限流）
- 每个子代理独立读取 references/ 获取检测知识
- 检测结果由父代理汇总，不修改原文件

### 子代理任务模板

每个子代理的 prompt 应包含：

```
你是创意与选题检查专家。请检测以下章节文件：

1. Read 读取章节文件：[章节文件路径]
2. Read 读取基础信息文件（如有）：[概念文件路径]
3. Read 读取检测知识：
   - skills/concept-check/references/check-dimensions.md（四维度检测标准）
   - skills/concept-check/references/optimization-formulas.md（优化公式）
   - skills/concept-check/references/report-template.md（报告模板）
4. 逐项扫描四个维度，定位问题
5. 按单章报告格式输出检测结果
6. 返回：综合评分、各维度得分、问题列表、优化建议
```

### 分批派发流程

```
章节列表 = [第1章, 第2章, ..., 第N章]
  ↓
按每批最多3章分组：
  批次1 = [第1章, 第2章, 第3章] → 并行派发3个 Task
  等待批次1全部完成
  ↓
  批次2 = [第4章, 第5章, 第6章] → 并行派发3个 Task
  等待批次2全部完成
  ↓
  ...直到所有章节检测完毕
```

---

## 第五步：汇总报告

所有章节检测完成后，将汇总报告写入小说目录下的 `review/concept-report.md`：

```markdown
## 批量创意与选题检查报告

### 总览

| 章节 | 评分 | 等级 | 最严重问题 | 建议操作 |
|------|------|------|------------|----------|
| 第1章 | 72/100 | 良好 | 辨识度不足 | 局部优化 |
| 第2章 | 45/100 | 及格 | 题材脱节 | 重点改写 |
| ... | ... | ... | ... | ... |

**平均分**：N/100
**最弱章节**：第X章（Y分）
**最常见问题**：[问题类型]

### 各章详情

[各子代理返回的单章报告]

### 全局建议

#### 共性问题
1. ...

#### 优先修改顺序
1. 第X章 → 原因
2. 第Y章 → 原因
```

---

## 注意事项

- 子代理使用 `general-purpose` 类型，model 使用 `sonnet`
- 每个子代理的 tools 需要 `Read`
- 如果某个子代理检测失败，记录错误并继续处理其他章节
- 检测是只读操作，不修改原文件
- 汇总报告写入 `[小说目录]/review/concept-report.md`
