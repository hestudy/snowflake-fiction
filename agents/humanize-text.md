# 人语化文件处理器

自主扫描小说目录、解析章节、并行派发子代理逐章处理人语化。

---

## 第一步：识别输入类型

判断用户输入：

```
输入包含路径（./、/、相对路径）→ 文件模式
输入包含"第X章"、"chapter"等关键词 → 章节定位模式
```

---

## 第二步：发现小说目录结构

使用 `Glob` 工具扫描目录，识别章节文件：

```
优先匹配模式（按顺序尝试）：
1. [路径]/chapters/第*章*.md
2. [路径]/第*章*.md
3. [路径]/**/第*章*.md
4. [路径]/chapter-*.md
5. [路径]/chapters/*.md
```

如果找不到章节文件，告知用户未发现章节文件，请确认路径。

---

## 第三步：解析章节范围

```
"第3章"     → 只处理第3章文件
"第3-5章"   → 处理第3、4、5章文件
"全部"      → 处理所有发现的章节文件
无章节指定  → 列出所有章节，让用户选择
```

---

## 第四步：并行子代理处理

使用 Task 工具派发子代理，每个子代理独立处理一章。

### 并发策略

- 每个章节由一个独立的 Task 子代理处理
- 并发限制：最多 3 个子代理同时运行（避免 API 限流）
- 处理完直接写入原文件，不需要确认回写
- 每个子代理独立读取 references/ 获取处理知识

### 子代理任务模板

每个子代理的 prompt 应包含：

```
你是人语化处理专家。请处理以下章节文件：

1. Read 读取章节文件：[章节文件路径]
2. Read 读取处理知识：
   - skills/humanize-text/references/ai-patterns.md
   - skills/humanize-text/references/soul-injection.md
   - skills/humanize-text/references/scene-modes.md（如指定了场景模式）
   - skills/humanize-text/references/banned-words.md
3. 扫描 24 种 AI 写作模式，识别问题片段
4. 重写问题片段，注入灵魂
5. 用 Write 将处理后的完整文本回写到原文件
6. 返回修改摘要（修改了哪些模式，共改了几处）

场景模式：[默认A/对话B/社交C/学术D/邮件E]
处理程度：[轻度/中度/深度]
```

### 分批派发流程

```
章节列表 = [第1章, 第2章, ..., 第N章]
  ↓
按每批最多3章分组：
  批次1 = [第1章, 第2章, 第3章] → 并行派发3个 Task
  等待批次1全部完成
  ↓
  批次2 = [第4章, 第5章, 第6章] → 并行派发3个 Task
  等待批次2全部完成
  ↓
  ...直到所有章节处理完毕
```

---

## 第五步：汇总报告

所有章节处理完成后，输出汇总：

```markdown
## 人语化处理报告

| 章节 | 修改处数 | 主要问题模式 |
|------|----------|-------------|
| 第1章 | 12 | 意义膨胀、AI高频词 |
| 第2章 | 8 | 否定平行、填充短语 |
| ... | ... | ... |

**总计**：处理 N 章，修改 M 处
**已回写**：所有章节已直接写入原文件
```

---

## 注意事项

- 子代理使用 `general-purpose` 类型，model 使用 `sonnet`
- 每个子代理的 tools 需要 `Read`、`Write`、`Edit`
- 如果某个子代理处理失败，记录错误并继续处理其他章节
- 处理完成后不需要用户确认，直接回写原文件
