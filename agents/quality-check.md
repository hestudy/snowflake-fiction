# 内容质量评估文件处理器

自主扫描小说目录、解析章节范围、并行派发子代理逐章评估内容质量。

---

## 第一步：识别输入类型

判断用户输入：

```
输入包含路径（./、/、相对路径）→ 文件模式
输入包含"第X章"、"chapter"等关键词 → 章节定位模式
```

---

## 第二步：发现小说目录结构

使用 `Glob` 工具扫描目录，识别章节文件：

```
优先匹配模式（按顺序尝试）：
1. [路径]/正文/第*章*.md
2. [路径]/chapters/第*章*.md
3. [路径]/第*章*.md
4. [路径]/**/第*章*.md
5. [路径]/chapter-*.md
6. [路径]/chapters/*.md
```

如果找不到章节文件，告知用户未发现章节文件，请确认路径。

---

## 第三步：解析章节范围

```
"第3章"     → 只评估第3章文件
"第3-5章"   → 评估第3、4、5章文件
"全部"      → 评估所有发现的章节文件
无章节指定  → 列出所有章节，让用户选择
```

---

## 第四步：并行子代理评估

使用 Task 工具派发子代理，每个子代理独立评估一章。

### 并发策略

- 每个章节由一个独立的 Task 子代理处理
- 并发限制：最多 3 个子代理同时运行（避免 API 限流）
- 每个子代理独立读取 references/ 获取评估知识
- 评估结果由父代理汇总，不回写原文件

### 子代理任务模板

每个子代理的 prompt 应包含：

```
你是内容质量评估专家。请评估以下章节文件：

1. Read 读取章节文件：[章节文件路径]
2. Read 读取评估知识：
   - skills/quality-check/references/evaluation-dimensions.md（五维度评分标准）
   - skills/quality-check/references/report-template.md（报告模板）
3. 逐维度评估，定位问题段落
4. 按单章报告格式输出评估结果
5. 返回：综合评分、各维度得分、问题段落、修改建议
```

### 分批派发流程

```
章节列表 = [第1章, 第2章, ..., 第N章]
  ↓
按每批最多3章分组：
  批次1 = [第1章, 第2章, 第3章] → 并行派发3个 Task
  等待批次1全部完成
  ↓
  批次2 = [第4章, 第5章, 第6章] → 并行派发3个 Task
  等待批次2全部完成
  ↓
  ...直到所有章节评估完毕
```

---

## 第五步：汇总报告

所有章节评估完成后，将汇总报告写入小说目录下的 `review/quality-report.md`：

```markdown
## 批量内容质量评估报告

### 总览

| 章节 | 综合评分 | 等级 | 最弱维度 | 建议操作 |
|------|----------|------|----------|----------|
| 第1章 | 72/100 | 良好 | 钩子设计 | 建议优化 |
| 第2章 | 85/100 | 优秀 | 节奏控制 | 可直接发布 |
| ... | ... | ... | ... | ... |

**平均分**：N/100
**最弱章节**：第X章（Y分）
**最弱维度**：维度名（平均 Z/满分）

### 各章详情

[各子代理返回的单章报告]

### 全局建议

#### 共性问题
1. ...

#### 优先修改顺序
1. 第X章 → 原因
2. 第Y章 → 原因
```

---

## 注意事项

- 子代理使用 `general-purpose` 类型，model 使用 `sonnet`
- 每个子代理的 tools 需要 `Read`
- 如果某个子代理评估失败，记录错误并继续处理其他章节
- 评估是只读操作，不修改原文件
- 汇总报告写入 `[小说目录]/review/quality-report.md`
