# 创作期续写 Skill

基于场景规划和人物宝典，逐章生成小说正文。每章生成后自动执行流水账自检。

## 使用时机

- 场景规划完成后，进入创作期
- 日更生产：每天续写N章
- 断更恢复：从最后一章继续
- 局部重写：重新生成指定章节

---

## 单章生成流程

1. 读取该章的场景规划（`08-场景规划/`）
2. 读取相关人物宝典（`06-人物宝典/`）
3. 读取前一章正文（保持连贯）
4. 读取 `./references/writing-guide.md` 获取写作提示词
5. 读取 `00-写作风格.md` 获取笔风参数（如文件不存在则使用默认风格）
6. 生成初稿（2000-4000字），严格遵守写作风格配置
7. 执行流水账自检（4维度）
8. 保存到 `正文/第N章.md`

---

## 生成要求

| 项目 | 要求 |
|------|------|
| 字数 | 2000-4000字/章 |
| 风格 | 严格遵守 `00-写作风格.md` 配置（如文件不存在则根据题材调整） |
| POV | 由写作风格中的叙事视角决定，保持视角人物一致 |
| 必须包含 | 核心事件、标志性语言/动作、情感锚点、结尾钩子 |
| 避免 | 信息倾倒、与人设不符、前后矛盾 |

---

## 流水账自检

每章生成后必须通过以下4维度自检：

### 1. 冲突检查
- 本章是否有明确冲突？（人物冲突/内心冲突/环境冲突）
- 冲突是否有解决或推进？
- 是否有对抗力量阻碍主角？

### 2. 情绪起伏检查
- 读者本章会感到什么情绪？（愤怒/紧张/期待/感动）
- 是否有情绪高潮点？
- 情绪是否有变化曲线？

### 3. 期待感检查
- 读者会想知道"接下来发生什么"吗？
- 是否设置了悬念或钩子？
- 是否有未完成的目标/任务？

### 4. 节奏检查
- 是否有事件发生？（不是单纯的"过了一天"）
- 事件是否有因果关系？
- 是否有紧张与舒缓的交替？

**流水账警告信号**：
- 连续3段以上只是"描述发生了什么"
- 没有任何角色的情绪反应
- 没有冲突或矛盾
- 读完后不记得发生了什么

自检不通过时，自动修正后再保存。

## 开篇强化

如果是第1-3章，额外执行开篇强化检查：

| 章节 | 检查点 |
|------|--------|
| 第1章 | 开篇第一句有冲击力？避免背景介绍式开场？首段制造冲突/危机/悬念？ |
| 第2章 | 核心设定通过剧情展开？金手指/机遇足够吸引？ |
| 第3章 | 主角展示金手指能力？有打脸/逆袭情节？结尾留有钩子？ |

---

## 章节结尾钩子

每章结尾必须设置钩子，详见 `./references/writing-guide.md`。

5种钩子类型：悬念钩、转折钩、期待钩、危机钩、揭秘钩。

## 输出格式

```markdown
# 第[X]章：[章节标题]

[正文内容]

---
**本章字数**：[X]
**情感锚点**：[...]
**结尾钩子**：[...]
```

---

## 前置检查

生成前必须确认：
1. 该章节的场景规划文件存在（`08-场景规划/`）
2. 人物宝典目录存在（`06-人物宝典/`）
3. 如非第1章，前一章正文已存在

缺少场景规划的章节跳过并报告，不强行生成。
