# 写作提示词与检查清单

本文件包含章节正文生成的提示词模板、流水账自检清单和钩子设计指南。

[本章出现的角色信息]

## 生成要求

1. **字数**：[2000-4000字]
2. **风格**：[根据题材调整]
3. **POV**：[视角人物]
4. **必须包含**：
   - 场景规划中的所有核心事件
   - 人物的标志性语言/动作
   - 本章的情感锚点
   - 结尾的悬念/钩子

5. **避免**：
   - 信息倾倒式对话
   - 与人设不符的行为
   - 前后矛盾的细节

## 流水账自检清单

### 1. 冲突检查
- [ ] 本章是否有明确的冲突？（人物冲突/内心冲突/环境冲突）
- [ ] 冲突是否有解决或推进？
- [ ] 是否有对抗力量阻碍主角？

### 2. 情绪起伏检查
- [ ] 读者本章会感到什么情绪？（愤怒/紧张/期待/感动）
- [ ] 是否有情绪高潮点？
- [ ] 情绪是否有变化曲线？

### 3. 期待感检查
- [ ] 读者会想知道"接下来发生什么"吗？
- [ ] 是否设置了悬念或钩子？
- [ ] 是否有未完成的目标/任务？

### 4. 节奏检查
- [ ] 是否有事件发生？（不是单纯的"过了一天"）
- [ ] 事件是否有因果关系？
- [ ] 是否有紧张与舒缓的交替？

**流水账警告信号**：
- 连续3段以上只是"描述发生了什么"
- 没有任何角色的情绪反应
- 没有冲突或矛盾
- 读完后不记得发生了什么

## 毒点自检（通用）

> 来源：番茄小说LV5作者鹿时七。详细标准见 `../../snowflake-fiction/references/genre-writing-guide.md` §6.11。

### 轻毒点检查
- [ ] 主角成长是否有合理过渡（不是突然变强）？
- [ ] 反派作恶后是否及时受到惩罚（有仇当场报，大Boss除外）？
- [ ] 人设是否前后一致（转变有铺垫和触发点）？
- [ ] 单个小剧情是否控制在1-2章内解决（不拖沓）？
- [ ] 换地图是否有必要且不过于频繁？

### 重毒点检查
- [ ] 主角是否避免了弱智行为推动剧情？
- [ ] 主角是否避免了圣母式无底线原谅？
- [ ] 主角是否有反击（不是总被虐）？
- [ ] 价值观是否正向（不反三观）？

### 人设过度检查
- [ ] 活泼角色是否过度调皮变蠢货？
- [ ] 感情戏是否"少说多做"（不过度深情变油腻）？
- [ ] 高冷角色是否过度变傲慢？
- [ ] 是否有说教/爹味输出（过度深刻）？
- [ ] 善良角色是否有底线（不过度善良变圣母）？

## 开篇强化检查

如果是小说开篇（第1-3章），需要额外检查：

### 第1章：3秒抓住读者
- [ ] 开篇第一句是否有冲击力？
- [ ] 是否避免了"背景介绍式"开场？
- [ ] 是否在首段就制造了冲突/危机/悬念？
- [ ] 信息密度是否合理？（避免信息轰炸）

### 第2章：世界观展开
- [ ] 核心设定是否通过剧情展开？
- [ ] 金手指/机遇是否足够吸引人？
- [ ] 是否避免了纯说明文式的世界观介绍？

### 第3章：第一个爽点
- [ ] 主角是否展示了金手指的能力？
- [ ] 是否有打脸/逆袭/获得认可的情节？
- [ ] 结尾是否留有钩子？

**开篇雷区警示**：

| 雷区 | 表现 | 修改方向 |
|------|------|----------|
| 寡淡开场 | "林风是个普通人..." | 从危机/冲突开始写 |
| 信息轰炸 | 三章内倾倒所有设定 | 设定通过剧情展开 |
| 拖沓入题 | 大量铺垫后才进正题 | 删减前奏，直接入戏 |

## 章节结尾钩子设计

每章结尾必须设置钩子，引导读者点击"下一章"：

### 常用钩子类型

1. **悬念钩**：留下未解之谜
   示例："他不知道的是，真正的危险才刚刚开始……"

2. **转折钩**：出乎意料的揭示
   示例："而那个杀手，竟然是他的亲生弟弟。"

3. **期待钩**：预告即将发生的大事
   示例："明天，就是宗门大比的日子。"

4. **危机钩**：制造紧迫感
   示例："门外传来脚步声——有人来了。"

5. **揭秘钩**：揭示重要信息
   示例："他终于知道了，父亲当年'失踪'的真相。"

### 钩子设计原则
- 钩子必须与主线剧情相关
- 不要"为钩而钩"，制造虚假悬念
- 钩子要在后续章节收束

### 悬念密度标准（崩文预防）

**量化指标**：每章至少保持 1.75 个活跃钩子

| 钩子数量 | 风险等级 | 表现 |
|---------|---------|------|
| ≥2个 | 安全 | 读者追更动力强 |
| 1.5-2个 | 合格 | 基本维持期待感 |
| 1-1.5个 | 警告 | 读者可能流失 |
| <1个 | 崩文风险 | 读者大量弃书 |

**钩子计算方式**：
- 章节结尾悬念 = 1个钩子
- 章节中段转折/危机 = 0.5个钩子
- 未解决的问题线 = 0.25个钩子/条

**悬念交替铺垫原则**：

在第1个悬念揭晓之前，就要铺垫好第2个悬念——不是快结束时才铺，而是在写第1个悬念的同时穿插第2个悬念相关的人或事物。

```
悬念1的故事中 → 穿插悬念2的部分故事 + 悬念3里人物的露面或提及
悬念2的故事中 → 穿插悬念3的部分故事 + 悬念4的露面或提及
一层层往下铺，丝滑切换
```

**ABC三个爽点滚动铺垫法**：

始终保持三个爽点剧情在手：A（当前写）、B（已铺垫）、C（正在构思）。写A时铺垫B、构思C，写B时铺垫C、构思D，无限循环。

```
写A剧情 → 铺垫B剧情 → 构思C剧情
写B剧情 → 铺垫C剧情 → 构思D剧情
写C剧情 → 铺垫D剧情 → 构思E剧情
……无限循环
```

示例：
- A剧情：主角炼制了一枚品相垃圾的丹药（实际是极品丹药）
- B剧情：送给公主当生辰宴礼物，反派嘲讽，验证后打脸
- C剧情：反派找主角报复
- D剧情：生辰宴上大家族天骄看中主角炼丹天赋，邀请合作

在写B剧情时，就在对话中引出反派和天骄，为C和D做铺垫。只要铺垫到位，一切剧情都会顺畅；没有铺垫，信息就只能靠旁白强行解释。

## 前30万字节奏里程碑

> 来源：番茄作家课堂·徐二家的猫（金番作者，代表作《请叫我鬼差大人》《从前有座镇妖关》）

百万长篇最舒适的前30万字节奏规划：

```
前3章：新奇切入点，把读者代入剧情
  ↓
4-5章：适当抛出核心设定
  ↓
3万字左右：短期目标式爆发点 + 为10万字大爆发埋伏笔
  ↓
3-10万字：分批次完善男主人设、世界观背景
  ↓ 通过一个接一个小事件，逐渐将氛围变得紧张、压抑
10万字：第一次大爆发
  ↓ 爆发后读者疲劳，重新起钩子 + 日常缓解
10-30万字：略微放缓节奏，每3-5万字一个情绪小高潮
  ↓ 添加有趣的新角色，每章都有"看点"
30万字：第二次大高潮
  ↓ 再次放缓，自然过渡到30-50万字剧情
```

**核心比喻**：像种植物——不要把所有种子一口气撒下去，分季节、分品种播种，一年四季都有收获。

**坑位管理法**：先挖个大坑，顺便挖俩小坑；填了小坑，再挖大坑小坑，顺便把第一个大坑填了……始终做到：**前方有坑，手中有铲，随时能挖，随时能填**。

### 爽点过密警告

新人作者最常犯的错误：前期把所有新颖有趣的剧情、爆发点、钩子一股脑甩上去，导致：

| 阶段 | 问题 | 后果 |
|------|------|------|
| 前期（0-10万字） | 爽点过密，爆发太多 | 读者看得很嗨，但作者思维储备快速消耗 |
| 中期（30-50万字） | 思维枯竭，被迫放慢 | 前后节奏落差过大，读者不适应 |
| 后期 | 强行调整节奏 | 读者主动弃书 |

**关键原则**：节奏一定要稳！宁愿牺牲一些前期的高密集爆发，也要给读者一种相对舒适的阅读环境。连续的高节奏剧情会导致阅读疲劳，爽点重叠会让读者觉得枯燥。

**自检**：写完前10万字后，回头检查——爽点分布是否均匀？是否还有足够的"弹药"支撑后续50万字？

## 情绪波浪公式（压→缓→压→爆发）

> 来源：番茄作家课堂·徐二家的猫（金番作者，代表作《请叫我鬼差大人》《从前有座镇妖关》）

剧情节奏的本质是**海浪**——不能一直压，也不能一直爽，要有起伏交替。

**核心公式**：
```
压抑（环境/困境塑造） → 缓解（希望/温情） → 再压（更强的打击） → 强爆发（情绪释放）
```

**示例（兄妹求医）**：
```
第一轮压：兄妹跋山涉水求医，被各家医馆拒绝
  ↓ 缓解：两人虽落魄但相依为命，充满希望，让人会心一笑
第二轮压：继续被赶出来，绝望加深
  ↓ 缓解：出现一位好心大夫，收进医馆（希望就是钩子）
第三轮压（极度）：大夫是人贩子，妹妹被拐，男主被打断双腿
  ↓ 强爆发：男主血洗医馆，读者情绪彻底释放
```

**关键原则**：
- 不能一直压——压久了读者上不来气，需要用温情/希望缓解
- 缓解本身就是钩子——读者在等待好的结果
- 大爆发前必须有极度压抑——善与恶的反差、生与死的跨越，把情绪压到极致
- 爆发后立刻埋新钩子——不给读者离开的机会

**节奏周期规划**：
```
前10万字：小压→小爆发 循环，积累到大爆发
10-20万字：过渡期，新冲突铺垫
30万字：第二次大爆发
```

**自检**：写完一段剧情后，画出情绪曲线——如果是一条平线，就是流水账；如果只有下降没有上升，读者会窒息。

## 字数预算法（节奏控制的笨办法）

> 来源：番茄作家课堂·徐二家的猫

列出大纲后，**强行给每段剧情分配字数预算**，然后严格按预算写。

**操作步骤**：
1. 列出接下来的剧情大纲（如：交代妹妹病情，展现两人现状）
2. 给这段剧情分配字数（如：4000字）
3. 按预算写，写超了就反思：
   - 是否有剧情繁琐化？
   - 是否有无效剧情？
   - 如果有，删减！
4. 删到实在不知道还能删哪里 = 你当前的文字控制极限
5. 按这个字数节奏规划后续剧情

**效果**：产出的剧情节奏稳定，不会过快也不会过慢，情绪有起伏、有压、有爆发。

**字数预算参考**：

| 剧情类型 | 建议字数 | 说明 |
|---------|---------|------|
| 日常铺垫/缓解 | 2000-3000字 | 不宜过长，避免拖沓 |
| 冲突升级/压抑 | 3000-5000字 | 需要足够篇幅积累情绪 |
| 高潮爆发 | 2000-4000字 | 集中爆发，不宜冗长 |
| 爆发后收尾+埋钩 | 1000-2000字 | 快速过渡到下一个循环 |

## 副本三段式结构

副本（一段完整的故事事件）从结构上分为三段，每段承担不同的爽点功能：

| 阶段 | 功能 | 爽点角色 | 核心任务 |
|------|------|----------|----------|
| 起因 | 铺垫和期待 | 爽点预热 | 丢饵，让读者模糊预见即将发生的事 |
| 经过 | 剧情和代入 | 爽点爆发 | 故事主干，埋好爽点逐步引爆进入高潮 |
| 结果 | 收获和成长 | 爽点回味 | 高潮延续，给读者获得感 |

### 起因：铺垫和期待

- 提前埋线，不要事到临头突然来事件
- 期待感的本质是"知而未见"——读者预见了即将发生的事，想看它落地，但你没让它落地，中间的线索却不断指向这件事
- "事"的本质是人物碰撞和角色变化，不是单纯的事件

### 经过：剧情和代入

- 代入每个角色视角去写，改文比写文更重要
- 适度合理、逻辑自洽即可，过度追求合理会卡文且缺失爽感
- 高潮篇幅不必太长，集中爆发比冗长紧张更有效
- 着重描写主角的闪光点（破局之策），让事件反哺人设
- 不断写小起伏，避免大高潮后期待感耗尽导致掉追读

### 结果：收获和成长

- 高潮后舒缓节奏，给读者"按摩"
- 角色必须有真正的收获：财富、道具、能力、情报线索、新人物关系
- 收获最好能成为下一个事件的钩子，与后续剧情有连接
- 穿插日常时，对话要展现角色对某件事的态度以强化人设，不写无意义水文

## 爽点节奏与倒推法

### 爽点设计原则

- 每个副本/剧情段必须先想出一个爽点，这个爽点就是模糊的大纲
- 爽点可以不是非常炸，但一定要有记忆点
- 不同类型爽点不同：日常文重收获，横推文重碾压打脸，玄幻重成长，悬疑重反转，恋爱重拉扯
- 爽点本质是反哺主角，让读者看到主角与众不同的出彩之处

### 倒推法创作流程

```
设计爽点（高潮/结局）→ 反推故事，填充枝节 → 开写
```

1. 把爽点当成剧情高潮，锚定在中后段位置
2. 从结局倒推整个故事，比从前往后想简单得多
3. 结局固定后人物线收束，更容易揣测角色动机
4. 倒推大纲可以大胆往前文塞伏笔，避免"先射箭再画靶"

### 大纲灵活性

- 大纲不要写得太细，避免填充式写作变成"做作业"
- 挖坑 = 拉期待，锚定结局后伏笔指向同一方向，即使结局不够精彩，伏笔同时引爆也有加成
- 找到最适合自己的细纲/粗纲平衡点

## 伏笔策略与锁链模式

### 伏笔深浅分类

| 类型 | 特点 | 适用场景 | 风险 |
|------|------|----------|------|
| 深伏笔 | 长线铺垫，引爆时爽感强 | 主线关键反转、角色秘密 | 太深可能被读者遗忘，白埋 |
| 浅伏笔 | 短平快，直接引爆 | 章节内小反转、即时打脸 | 爽感持续时间短 |

### 锁链模式（事件嵌套铺垫）

不要用串联模式 A→B→C→D 排队写，要像锁链一样，下一个事件的开头嵌入前一个事件的过程中：

```
A1(起因) → A2(经过) → B1(起因) → A3(结果) → B2(经过) → C1(起因) → B3(结果)...
```

- 在第一个事件没结束时就引出下一个事件，提前埋伏笔做期待
- 铺垫时可以适当"谜语人"：侧写配角的神态反应小动作，让人一看就知道有鬼，但不明说
- 伏笔来源：新消息、新情报、新人物、传言八卦、神秘发现

### 多剧情嵌套（进阶）

锁链模式熟练后，可形成多剧情铺垫：

```
A故事中段 → 引出B故事
A故事末尾 → 捎带提一嘴C故事
B故事开始 → C故事造势
B故事完结 → D故事跟上
```

注意：嵌套不是越多越好，要平衡故事和技巧，过于花里胡哨会引起阅读疲惫。

## 时间线类型写作技巧

如果 `00-一句话概括.md` 中记录了非顺叙时间线类型，正文写作需要额外注意以下技巧。

### 圆环时间线写作技巧

**时间线切换技巧（感官触发）**：

切换时间线不能生硬地写「三年前」，要用感官触发自然过渡：

```
❌ 生硬切换：
三年前的那个夏天，林晓还在读大学。

✅ 感官触发：
咖啡的苦味在舌尖散开——和三年前那杯一模一样。
那时候她还在学校旁边的小店里，对面坐着的人还没有消失。
```

常用感官锚点：
- 气味（咖啡、雨后泥土、某种香水）
- 声音（一首歌、雨声、某句话）
- 触感（一件旧衣服、一枚硬币）
- 视觉（一张照片、一个相似的场景）

**信息差管理**：

圆环时间线的核心魅力是信息差——读者在不同时间线中拼凑真相。

| 时间线 | 信息策略 |
|--------|----------|
| 现在线 | 展示结果，隐藏原因（读者想知道「为什么」） |
| 过去线 | 揭示原因，制造新疑问（回答一个问题，抛出两个新问题） |
| 未来线 | 暗示走向，保留悬念（让读者猜测但不确定） |

### 读档制写作技巧

**重复场景的处理（递减篇幅）**：

同一个场景在不同循环中出现时，篇幅必须递减：

```
第1次（详写，2000字）：
林晓推开门，走进那间熟悉的办公室。窗帘半拉着，阳光从缝隙里挤进来，
在地板上画出一道歪歪扭扭的线。桌上的咖啡还冒着热气，旁边放着一份
没拆封的快递。她注意到墙上的钟指向三点十五分……

第3次（略写，500字）：
又是这间办公室。三点十五分，咖啡冒热气，快递没拆。
但这次她没看这些——她直接走向书架第三层，抽出那本蓝色笔记本。

第5次（一笔带过，100字）：
第五次推开这扇门。她闭着眼都能走到书架前。
```

**情绪递进模板**：

```
第1轮：好奇（这是怎么回事？）
第2轮：兴奋（我可以利用这个！）
第3轮：焦虑（为什么还是失败？）
第4轮：绝望（我永远出不去了……）
第5轮：顿悟（原来问题在我自己）
第6轮：突破（这次不一样了）
```

### 纯循环写作技巧

**循环感营造**：

用重复的句子作为循环标记，让读者一看就知道「又开始了」：

```
每个循环的开头都用同一句话：
「闹钟响了。六点整。窗外是灰蒙蒙的天。」

但每次循环中加入微小变化：
第1次：「闹钟响了。六点整。窗外是灰蒙蒙的天。」
第3次：「闹钟响了。六点整。窗外是灰蒙蒙的天。——等等，今天的云好像少了一朵。」
第7次：「闹钟响了。六点整。窗外……是晴天。」
```

**微小变化的递进**：
- 前期：环境细节变化（天气、物品位置）
- 中期：人物反应变化（NPC说了不同的话）
- 后期：规则变化（循环时间缩短、记忆开始模糊）

### 多线并行写作技巧

**切换标记**：

每条线用固定的开头标记，帮助读者快速定位：

```
A线（主角视角）：总是以动作开头
「林晓踢开门，冲进雨里。」

B线（反派视角）：总是以环境开头
「办公室里很安静，只有钟表的滴答声。」

C线（配角视角）：总是以对话开头
「"你确定要这么做？"」
```

**语感差异化**：

不同线的叙事语感要有区别，让读者不看标记也能分辨：

| 线 | 语感特点 | 句式 |
|----|----------|------|
| 主角线 | 紧张、急促 | 短句为主，多动作 |
| 反派线 | 冷静、从容 | 长句为主，多心理 |
| 配角线 | 温暖、犹豫 | 对话为主，多情感 |

**切换频率控制**：

| 阶段 | 切换频率 | 原因 |
|------|----------|------|
| 开篇（前5章） | 每2-3章切换 | 让读者先熟悉每条线 |
| 发展期 | 每1-2章切换 | 加快节奏，制造交织感 |
| 高潮前 | 每场景切换 | 多线加速碰撞 |
| 高潮 | 各线汇合，不再切换 | 集中爆发 |

## 换地图场景处理（崩文预防）

### 换地图的两大雷区

| 雷区 | 表现 | 后果 |
|------|------|------|
| 背景堆砌 | 大段描写新地图的地理/历史/势力 | 读者跳过或弃书 |
| 故事断裂 | 换地图后主线冲突消失 | 读者失去追更动力 |

### 正确的换地图公式

```
换地图 = 主线冲突延续 + 新地图元素服务冲突
```

**错误示范**：
```
❌ 主角来到北境，这里常年冰雪覆盖，有三大家族统治，分别是...（纯背景介绍）
```

**正确示范**：
```
✅ 主角刚踏入北境，追杀他的杀手就到了——这里是杀手的地盘。（冲突延续）
   "在北境，我们有规矩。"杀手冷笑，"外来者，死。"（通过冲突展示设定）
```

### 换地图检查清单

- [ ] 换地图的原因是否与主线冲突相关？
- [ ] 新地图的设定是否通过剧情展开（而非背景说明）？
- [ ] 主角在新地图的目标是否明确？
- [ ] 是否避免了大段环境/势力介绍？

### 新手建议

- 前30章建议聚焦单一地图，把主线冲突在一个地图内展开完整
- 实在要换地图尽量往后拖，字数越长弃文概率越低
- 校园文全本在学校，宅斗文全本在宅院——把故事线集中在一个地方

## 金手指设计与情节推动

### 金手指创新设计原则

金手指不是固定的系统面板或戒指小老头，而是任何能推动剧情的独特设定。

**创新公式**：`[常见题材] + [非常规能力形态] = 有辨识度的金手指`

| 类型 | 老套写法 | 创新写法 | 案例 |
|------|---------|---------|------|
| 系统类 | 开局大面板强行绑定 | 极简触发、关键时刻出现 | 《神序001》脑海中的感知异能，出现少但每次推动剧情 |
| 穿越类 | 直接获得超强能力 | 能力是间接的、需要借力 | 《开局上交修仙界》金手指是现代文明本身，通过双鱼玉佩借力 |
| 升级类 | 一步到位无敌碾压 | 循序渐进、每次升级有期待 | 《序列公路》破二八大杠→三轮车→汽车，每次升级给读者新期待 |
| 生活类 | 神豪系统直接发钱 | 能力有条件、有趣味 | 《相亲就能变强》每次相亲获得奖励，条件本身就是剧情 |
| 反向类 | 金手指助力主角欲望 | 金手指与主角欲望相反，制造核心矛盾 | 想逃离皇朝的王子获得成帝系统；想赚钱的重生者获得亏钱系统 |

**反向金手指设计**：金手指作为欲望的负价值，是一种高辨识度的设计思路——主角想要A，金手指偏偏推着他走向B。这种内在矛盾本身就是故事引擎，无需额外制造冲突。

### 金手指展示节奏

```
❌ 错误：第一页就弹出大面板 → 读者索然无味
✅ 正确：通过剧情自然引出 → 读者和主角一起"发现"金手指
```

**分阶段展示策略**：

| 阶段 | 章节范围 | 展示内容 | 目的 |
|------|---------|---------|------|
| 初现 | 第1-3章 | 金手指首次触发，展示基础能力 | 建立期待 |
| 探索 | 第4-15章 | 逐步发现能力边界和限制 | 维持好奇 |
| 升级 | 每10-20章 | 解锁新能力或突破限制 | 制造爽点 |
| 危机 | 高潮前 | 金手指失效或不够用 | 制造张力 |

### 金手指推动中后期情节

金手指不是工具，是剧情引擎。每次金手指的升级/限制都应该制造新冲突：

```
金手指升级 → 新能力带来新问题 → 新冲突推动剧情
金手指限制 → 主角必须另寻出路 → 配角/智谋登场
金手指代价 → 使用有副作用 → 内心挣扎+外部压力
```

**避免两个极端**：
- 金手指太强 → 无敌碾压，读者失去期待感
- 金手指太弱 → 形同虚设，读者忘记它的存在

**平衡公式**：`金手指强度 = 刚好解决当前困境 + 无法应对下一个威胁`

## 情绪传递链与情绪递进

### 核心原理

剧情推动的本质是**情绪传递→情绪发泄→情绪递进**的循环。不要写事件本身，要写事件引发的情绪如何在角色之间传递和升级。

```
角色A受到冲击 → 情绪传递给身边的角色B/C
→ 角色A发泄情绪（行动）→ 情绪传递给对手D
→ 对手D发泄情绪（反击）→ 情绪升级回到角色A
→ 循环递进，直到最终爆发
```

### 情绪传递链示例（退婚剧情）

```
女配退婚 → 主角感到羞辱
  → 情绪传递：主角的愤怒传递给身边的父亲/家人
  → 情绪发泄：主角撕毁婚书
    → 情绪传递：女配生气，传递给身边的天才追求者
    → 天才发泄：亮出身份蔑视主角（情绪升级——屈辱更大）
      → 主角含怒出手，打不过被揍（情绪再次递进）
      → 对方挑衅："你永远没资格"（情绪又一次递进）
        → 主角获得金手指（情绪转折）
        → 宗门考核击败天才（情绪总爆发）
          → 天才羞辱传递给内门的哥哥（新一轮循环开始）
```

### 情绪递进的注意事项

- **递进要匹配**：主角遭遇的人物和事件要逐步升级（逼格更高、事情更大）
- **不能跳级**：前面都是小剧情，突然来一个人前显圣大爽点，再回到小剧情就会崩
- **开局不能太高**：开局就是宏大情绪（如镇守千年归来家族被屠），后续情绪递进空间会很小
- **多线汇聚更爽**：多条情绪线同时递进，最后一起爆发，比单线递进更有冲击力

### 不公平+对比情绪公式

> 来源：番茄签约作者任欢游（300万字创作经验）的实战方法论。
> 核心理念：挑起读者情绪最高效的方式不是单纯的压迫，而是**不公平待遇 + 对比反差**的叠加。

**公式**：`不公平（别人都有的我没有）+ 对比（同样的人却得到截然不同的待遇）= 情绪强烈递进`

**示例**：
```
基础版：男主要求女主AA制，连5块钱冰淇淋都要女主付2块5
  → 读者感受：有点抠门

升级版（+不公平+对比）：
男主要求女主AA制，连5块钱冰淇淋都要女主付2块5
  → 但他背地里给白月光花8800买演唱会VIP票，还带冰淇淋爆米花，全款
  → 读者感受：愤怒值拉满
```

**设计方法**：
1. 先写主角遭受的不公平待遇（被忽视/被苛待/被剥夺）
2. 紧接着展示对比对象获得的优待（被偏爱/被珍视/被给予）
3. 两者的反差越具体、越细节，情绪冲击越强

**适用场景**：
- 开篇代入感建立（让读者快速与主角共情）
- 情绪下行段落（用不公平制造压抑，为后续爆发蓄力）
- CP感情线（偏爱细节衬托法的反面——展示"不被偏爱"的痛）

**与情绪波浪公式的配合**：
```
不公平+对比（压抑蓄力）→ 缓解（一丝希望）→ 更大的不公平+对比（极度压抑）→ 主角爆发（情绪释放）
```

## 战斗场面的目的论写法

### 核心原则

战斗好不好看，不在于打斗的精彩程度，而在于**打斗要传递几种情绪**。战斗目的决定战斗篇幅和写法。

### 根据情绪传递数量决定战斗写法

| 传递情绪数 | 战斗篇幅 | 写法 | 示例 |
|-----------|---------|------|------|
| 1种 | 一招解决 | 快速干脆 | 击败恶徒→恶徒传递情绪给老大（引出下一段剧情） |
| 2种 | 越级挑战 | 展现天赋 | 救美女→①美女感激 ②美女震惊天赋→邀请参加家族比武 |
| 3种+ | 详细战斗 | 多层展示 | 救美女→①感激 ②震惊天赋 ③施展的武技是家族失传绝学→开启祖地 |
| 多种复杂 | 生死决斗 | 欲扬先抑 | 围观者情绪+对手情绪+底牌爆发+阵营对比→集中爆发 |

### 无目的战斗 = 水文字

```
❌ 无目的：反派嘲讽主角 → 主角打脸 → 然后就没有然后了
   → 对读者而言就是水文字，写得再精彩也没意义

✅ 有目的：反派嘲讽主角 → 主角打脸 → 反派传递情绪给更强的靠山
   → 靠山对后续剧情有推动作用 → 战斗服务于情节推进
```

### 生死决斗为什么好看

生死决斗之所以精彩，是因为传递了多种复杂情绪：
- 围观者：惊叹双方实力、主角阵营担忧、对手阵营得意
- 对手：从轻蔑→认真→震惊→恐惧
- 主角：隐忍→爆发底牌→情绪总爆发
- 铺垫的底牌施展出去，本身就是一种情绪爆发

### 战斗场景自检

- [ ] 这场战斗要传递几种情绪？
- [ ] 战斗篇幅是否匹配情绪传递数量？
- [ ] 战斗结束后是否推动了后续剧情？
- [ ] 是否避免了"打完就完"的无目的战斗？

## 五感代入写法

> 来源：番茄作家课堂·煮熟的来福鸽（800万字创作经验）

五感描写（视觉、听觉、嗅觉、触觉、味觉）是增强代入感最直接的文笔手段。读者会根据文字自动在脑海中构建画面和感觉。

**核心原则**：每个重要场景至少加入 1-2 种五感描写，让读者"身临其境"。

**五感代入示例**：

| 感官 | 示例 | 效果 |
|------|------|------|
| 触觉 | 潮湿的短袖黏在后背上 | 读者立刻感受到闷热 |
| 嗅觉 | 医院消毒水的味道 | 读者瞬间联想到医院场景 |
| 听觉 | 雨水砸在铁皮瓦上的声音 | 读者脑中自动播放声音 |
| 嗅觉 | 大雨过后泥土的味道 | 唤起读者的真实记忆 |
| 触觉 | 鱼鳞滑腻的触感 | 具象化的身体感受 |
| 味觉 | 吃果冻的口感 | 调动味觉记忆 |
| 视觉 | 夕阳的火烧云 | 画面感最强的感官 |

**使用技巧**：
- 优先选择读者日常生活中真实接触过的感官体验（熟悉感 = 代入感）
- 不需要五感全上，1-2种即可，过多反而拖节奏
- 五感描写要服务于情绪：紧张场景用触觉（心跳、手抖），温馨场景用嗅觉（饭菜香、花香）
- 通过主角的五感感受来构建环境，而非上帝视角的客观描述

**自检**：写完一个场景后，问自己——读者能"看到/听到/闻到/摸到/尝到"什么？如果答案是"什么都没有"，补一两句五感描写。
