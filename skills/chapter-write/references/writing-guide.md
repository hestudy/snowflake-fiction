# 写作提示词与检查清单

本文件包含章节正文生成的提示词模板、流水账自检清单和钩子设计指南。

---

## 单章生成提示词

```
请基于以下材料生成第 [X] 章正文：

## 输入材料

**章节标题**：[...]
**场景规划**：
[该章节所有场景的详细规划]

**人物宝典**（相关角色）：
[本章出现的角色信息]

**前文摘要**：
[前一章的结尾和关键信息]

**伏笔检查**：
[本章需要埋设或收束的伏笔]

---

## 生成要求

1. **字数**：[2000-4000字]
2. **风格**：[根据题材调整]
3. **POV**：[视角人物]
4. **必须包含**：
   - 场景规划中的所有核心事件
   - 人物的标志性语言/动作
   - 本章的情感锚点
   - 结尾的悬念/钩子

5. **避免**：
   - 信息倾倒式对话
   - 与人设不符的行为
   - 前后矛盾的细节

---

## 输出格式

# 第[X]章：[章节标题]

[正文内容]
```

文件中只保存章节标题和正文，不包含元数据。

子代理返回给父代理的摘要（不写入文件）：
```
章节：第[X]章
标题：[章节标题]
字数：[X]
核心事件：[...]
情感锚点：[...]
结尾钩子类型：[...]
```

## 流水账自检清单

### 1. 冲突检查
- [ ] 本章是否有明确的冲突？（人物冲突/内心冲突/环境冲突）
- [ ] 冲突是否有解决或推进？
- [ ] 是否有对抗力量阻碍主角？

### 2. 情绪起伏检查
- [ ] 读者本章会感到什么情绪？（愤怒/紧张/期待/感动）
- [ ] 是否有情绪高潮点？
- [ ] 情绪是否有变化曲线？

### 3. 期待感检查
- [ ] 读者会想知道"接下来发生什么"吗？
- [ ] 是否设置了悬念或钩子？
- [ ] 是否有未完成的目标/任务？

### 4. 节奏检查
- [ ] 是否有事件发生？（不是单纯的"过了一天"）
- [ ] 事件是否有因果关系？
- [ ] 是否有紧张与舒缓的交替？

**流水账警告信号**：
- 连续3段以上只是"描述发生了什么"
- 没有任何角色的情绪反应
- 没有冲突或矛盾
- 读完后不记得发生了什么

## 开篇强化检查

如果是小说开篇（第1-3章），需要额外检查：

### 第1章：3秒抓住读者
- [ ] 开篇第一句是否有冲击力？
- [ ] 是否避免了"背景介绍式"开场？
- [ ] 是否在首段就制造了冲突/危机/悬念？
- [ ] 信息密度是否合理？（避免信息轰炸）

### 第2章：世界观展开
- [ ] 核心设定是否通过剧情展开？
- [ ] 金手指/机遇是否足够吸引人？
- [ ] 是否避免了纯说明文式的世界观介绍？

### 第3章：第一个爽点
- [ ] 主角是否展示了金手指的能力？
- [ ] 是否有打脸/逆袭/获得认可的情节？
- [ ] 结尾是否留有钩子？

**开篇雷区警示**：

| 雷区 | 表现 | 修改方向 |
|------|------|----------|
| 寡淡开场 | "林风是个普通人..." | 从危机/冲突开始写 |
| 信息轰炸 | 三章内倾倒所有设定 | 设定通过剧情展开 |
| 拖沓入题 | 大量铺垫后才进正题 | 删减前奏，直接入戏 |

## 章节结尾钩子设计

每章结尾必须设置钩子，引导读者点击"下一章"：

### 常用钩子类型

1. **悬念钩**：留下未解之谜
   示例："他不知道的是，真正的危险才刚刚开始……"

2. **转折钩**：出乎意料的揭示
   示例："而那个杀手，竟然是他的亲生弟弟。"

3. **期待钩**：预告即将发生的大事
   示例："明天，就是宗门大比的日子。"

4. **危机钩**：制造紧迫感
   示例："门外传来脚步声——有人来了。"

5. **揭秘钩**：揭示重要信息
   示例："他终于知道了，父亲当年'失踪'的真相。"

### 钩子设计原则
- 钩子必须与主线剧情相关
- 不要"为钩而钩"，制造虚假悬念
- 钩子要在后续章节收束
