# 流水账检测四维度标准

## 维度1：无冲突检测（权重 30%）

### 检测清单

- [ ] 本章是否有明确的冲突/矛盾？
- [ ] 主角是否面临需要解决的问题？
- [ ] 是否有人物之间的对立/碰撞？
- [ ] 冲突是否足够强烈？

### 流水账表现

```
❌ "他起床，刷牙，吃早饭，出门上班，坐地铁，到公司打卡..."
   → 纯时间线推进，没有意外，没有阻碍
```

### 正确做法

```
✅ "他刚刷完牙，手机响了——房东的催租电话。
   他看了一眼余额：382块。房租要2000。"
   → 立刻出现需要解决的问题
```

### 评分标准

| 分数 | 描述 |
|------|------|
| 9-10 | 冲突强烈且多层次 |
| 7-8 | 有明确冲突 |
| 5-6 | 冲突存在但较弱 |
| 3-4 | 冲突模糊 |
| 1-2 | 几乎无冲突 |

---

## 维度2：无情绪检测（权重 25%）

### 检测清单

- [ ] 是否有角色的情绪变化？
- [ ] 读者是否能感受到角色的感受？
- [ ] 每段是否有情绪锚点？
- [ ] 是否有感官描写（视觉/听觉/触觉）？

### 流水账表现

```
❌ "他走进房间，看到桌上的信，打开看了一遍，然后放下。"
   → 纯动作，不知道他在想什么，感受如何
```

### 正确做法

```
✅ "他走进房间，看到桌上的信——是他等的那个offer。
   手抖了一下，信封差点掉地上。
   撕开信封的时候，他甚至不敢呼吸。"
   → 情绪层层递进
```

### 评分标准

| 分数 | 描述 |
|------|------|
| 9-10 | 情绪饱满且层次丰富 |
| 7-8 | 有明确情绪表达 |
| 5-6 | 情绪存在但单薄 |
| 3-4 | 偶有情绪点缀 |
| 1-2 | 纯动作描述，无情绪 |

---

## 维度3：无期待感检测（权重 25%）

### 检测清单

- [ ] 读者是否想知道接下来发生什么？
- [ ] 章节结尾是否有钩子/悬念？
- [ ] 是否有未解决的问题吸引读者？
- [ ] 是否有伏笔让读者好奇？

### 流水账表现

```
❌ "这就是他的一天。第二天，他又重复了同样的生活。"
   → 读者完全不想知道后面
```

### 正确做法

```
✅ "他不知道的是，那封信里藏着改变他命运的机会。
   而现在，他正准备把它扔进垃圾桶。"
   → 读者：别扔！快捡回来！
```

### 评分标准

| 分数 | 描述 |
|------|------|
| 9-10 | 强烈悬念，欲罢不能 |
| 7-8 | 有明确期待感 |
| 5-6 | 有些好奇但不强烈 |
| 3-4 | 期待感微弱 |
| 1-2 | 完全无期待感 |

---

## 维度4：信息堆砌检测（权重 20%）

### 检测清单

- [ ] 是否有大段背景/设定灌输？
- [ ] 信息是否通过互动展示？
- [ ] 信息密度是否合理？
- [ ] 是否有过渡拖沓？

### 流水账表现

```
❌ "青云宗是玄天大陆东南方最大的宗门，有三千年历史，
   创始人是化神期大能青云真人，宗门分为外门、内门、
   核心弟子，外门弟子有五千人..."
   → 像读说明书一样枯燥
```

### 正确做法

```
✅ ""外门弟子？"林默捏着那块粗糙的木牌，"就这？"
   接待的师兄冷笑："就这？你知道有多少人挤破头都进不来吗？"
   → 通过角色互动展示信息
```

### 评分标准

| 分数 | 描述 |
|------|------|
| 9-10 | 信息融入叙事，毫无堆砌感 |
| 7-8 | 信息展示自然 |
| 5-6 | 有些信息堆砌但可接受 |
| 3-4 | 明显信息灌输 |
| 1-2 | 大段说明文式堆砌 |

---

## 常见流水账场景

| 场景 | 流水账表现 | 修复方案 |
|------|------------|----------|
| 日常描写 | 起床刷牙吃早饭 | 加入意外打断 |
| 路途描写 | 坐地铁走路到公司 | 用内心戏/回忆填充 |
| 背景介绍 | 大段世界观说明 | 通过对话/冲突展示 |
| 时间过渡 | 三天过去了 | 用事件锚定时间 |
| 战斗描写 | 一招一式平铺 | 加入心理博弈，根据战斗目的决定篇幅 |
| 换地图 | 大段描写新地图的地理/历史/势力 | 通过冲突展示新地图规则 |

---

## 换地图场景的流水账预防

**高危场景**：换地图时最容易出现信息堆砌

| 场景类型 | 流水账表现 | 正确做法 |
|---------|-----------|---------|
| 换地图 | 大段描写新地图的地理/历史/势力 | 通过冲突展示新地图规则 |
| 新势力登场 | 介绍势力的组织架构和历史 | 通过对话/冲突展示势力特点 |
| 新能力获得 | 详细说明能力的原理和限制 | 通过使用过程逐步揭示 |

### 换地图检查清单

- [ ] 是否避免了"主角来到XX，这里是..."式开场？
- [ ] 新地图的设定是否通过剧情冲突展示？
- [ ] 是否有主线冲突在新地图延续？
- [ ] 信息密度是否控制在每段不超过2个新设定？

### 示例对比

```
❌ 信息堆砌（流水账）：
主角来到北境，这里常年冰雪覆盖，气温在零下30度左右。
北境由三大家族统治，分别是雪狼家族、冰熊家族和霜鹰家族。
雪狼家族掌控军事，冰熊家族掌控商业，霜鹰家族掌控情报...
（纯背景说明，读者会跳过）
```

```
✅ 冲突展示（正确）：
主角刚踏入北境城门，一支箭就射在脚边。
"外来者，报上名来。"城墙上的守卫冷喝。
主角举起双手："我只是路过。"
"路过？"守卫冷笑，"在北境，没有'路过'这个词。要么加入一个家族，要么——死。"
（通过冲突展示北境规则，读者不会跳过）
```
