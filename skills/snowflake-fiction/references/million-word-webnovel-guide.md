# 百万级网文创作指南

专为100万字以上的商业网文设计，涵盖平台规则、商业节奏、持续更新策略等。

---

## 百万级 vs 传统长篇

| 维度 | 传统长篇（10-50万字） | 百万级网文 |
|------|----------------------|------------|
| **更新节奏** | 完结后发布 | 日更2章（4000-6000字/天） |
| **创作周期** | 2-6个月 | 1-3年持续更新 |
| **章节数** | 50-200章 | 1000-3000章 |
| **结构** | 多卷（5-10卷） | 多地图/多阶段（每阶段20-50万字） |
| **读者互动** | 无 | 评论反馈、数据调整 |
| **商业节点** | 无 | 黄金三章、付费点、首订 |
| **爽点密度** | 章节级 | 每3章一个小高潮 |

## 商业网文核心节奏

### 黄金三章（决定读者去留）

黄金三章是网文最关键的三个章节，直接决定读者是否会继续阅读。

#### 第1章：开篇即冲突，3秒抓住读者

**核心原则**：从"炸楼"开始写，让读者立刻紧张起来。

**三大开篇雷区**：
| 雷区 | 问题 | 示例 |
|------|------|------|
| **寡淡开场** | 平平无奇，没吸引力 | "林风是个普通上班族，每天朝九晚五……" |
| **信息轰炸** | 大量设定倾倒 | "在玄天大陆东南方，有个三千年历史的青云宗……" |
| **拖沓入题** | 迟迟不进入正题 | 先写三段天气描写、再写两段起床洗漱 |

**正确示范**：
```
被公司开除的第三天，林风发现银行卡里只剩下382块钱。

手机响了——是他欠了三个月房租的房东。

"林风，再不交房租，明天就给我搬出去！"
```

**开篇切入点选择**：

| 切入方式 | 特点 | 适用场景 | 示例 |
|----------|------|----------|------|
| **剧情切入** | 直接冲突，节奏快 | 大多数爽文 | "你就是个废物！"分手协议摔在脸上 |
| **场景切入** | 氛围渲染，代入感强 | 悬疑、仙侠 | 雨夜破庙，主角带伤独坐 |

**新手推荐**：剧情切入（更直接，更易上手）

---

#### 第2章：世界观展开，主角获得金手指/机遇

**任务清单**：
- [ ] 揭示故事核心设定（系统/穿越/异能）
- [ ] 主角获得改变命运的机会
- [ ] 铺垫第一个小目标

**要点**：
- 设定通过剧情展开，不要大段说明文
- 让读者和主角一起"发现"金手指
- 金手指要足够吸引人

**示例**：
```
× 说明式："他获得了一个可以兑换万物的系统。"
✓ 剧情式："苏云脑海里的声音正在说话——【检测到宿主遭受羞辱，怒气值+100，炼丹系统激活……】"
```

---

#### 第3章：第一个爽点，小试牛刀

**任务清单**：
- [ ] 主角使用金手指解决小问题
- [ ] 打脸/逆袭/获得认可
- [ ] 结尾留钩子，引导点击"加入书架"

**爽点设计公式**：
```
铺垫（被看不起） → 反转（展示实力） → 反应（围观群众震惊）
```

**钩子类型**：

| 钩子类型 | 示例 |
|----------|------|
| 悬念钩 | "他不知道的是，真正的危险才刚刚开始……" |
| 转折钩 | "而那个杀手，竟然是他的亲生弟弟。" |
| 期待钩 | "明天，就是他突破筑基期的日子。" |
| 危机钩 | "门外传来脚步声——有人来了。" |

**第3章结尾必留钩子**，这是读者决定是否"加入书架"的关键时刻！

### 付费卡点（番茄：约30章，起点：约20章）

**付费前要完成的任务**：
- [ ] 主角人设立住
- [ ] 核心爽点展示完毕
- [ ] 至少2-3个重要配角登场
- [ ] 第一个大剧情或冲突
- [ ] 读者对后续产生强烈期待

**付费卡点设计**：
```
第X章结尾：
主角即将获得重大突破/进入新地图/面临大危机...
→ 读者必须付费才能看到结果
```

### 爽点密度设计

| 间隔 | 类型 | 示例 |
|------|------|------|
| **每3章** | 小爽点 | 打脸小反派、获得小奖励、能力提升 |
| **每10章** | 中爽点 | 解决一个小危机、获得重要物品、进入新场景 |
| **每30章** | 大爽点（小高潮） | 完成一个副本、击败阶段性Boss、身份蜕变 |
| **每100章** | 超大爽点（大高潮） | 地图切换、境界突破、彻底逆袭 |
| **每200-300章** | 换地图/换阶段 | 世界观扩展、新的起点 |

## 不要给读者出选择题

> 来源：番茄作家课堂·徐二家的猫（金番作者，代表作《请叫我鬼差大人》《从前有座镇妖关》）

百万字长篇保住读完率的关键原则：**把剧情设计成"判断对错题"，而非"选择题"**。

### 什么是"选择题"

让读者对主角的行为产生分歧的剧情设计：

```
❌ 选择题示例：
仇家杀了男主全家，男主复仇杀了对方，却发现对方还有一个善良的孩子……
→ 杀还是不杀？
→ 读者分裂：有人说该杀，有人说不该杀，有人说收养
→ 无论主角怎么选，都有一批读者弃书
```

### 什么是"判断对错题"

让所有读者对主角的行为达成共识的剧情设计：

```
✅ 判断题示例：
A杀了男主全家，横霸乡里，无恶不作 → 杀了他，对不对？对！
B倒卖人口，背信弃义，人神共愤 → 让他死，对不对？对！
→ 反派就是罪无可赦的反派，不要塑造悲歌，不要强行让文有深度
→ 杀了不会有任何读者流失
```

### 选择题的危害

| 选择题数量 | 读者流失 | 说明 |
|-----------|---------|------|
| 5道 | ~30% | 每次都有少数人不满 |
| 10道 | ~50% | 累积效应显著 |
| 20道 | ~80% | 即使每次站在多数人那边也留不住 |

### 常见选择题陷阱

| 陷阱 | 示例 | 修复方案 |
|------|------|----------|
| 反派洗白 | 反派有苦衷/有善良的家人 | 反派就是纯粹的恶人，不留同情空间 |
| 感情两难 | 女主父亲是杀父仇人，选爱情还是复仇？ | 仇人是共同的敌人，男女主一起报仇 |
| 道德困境 | 救一个人还是救一群人？ | 设计成主角有能力两全的局面 |

### 自检清单

- [ ] 本章主角的关键决定，是否所有读者都会认同？
- [ ] 反派是否有可能引发读者同情？（如果有，删掉那些同情点）
- [ ] 是否存在"怎么选都有人不满"的剧情？（如果有，重新设计）
- [ ] 感情线是否有让读者站队的风险？

### 例外情况

"判断对错题"原则适用于大多数商业网文。如果你的笔力足够强，可以驾驭道德灰色地带——但这是悬崖上跳舞，新人不建议尝试。

## 注意事项

1. **永远保持存稿**：网文最忌讳断更
2. **黄金三章决定生死**：前3章必须精雕细琢
3. **爽点密度不能降**：读者随时可能弃书
4. **伏笔要有收束**：但可以埋得很深（500章后收）
5. **人语化必须做**：平台AI检测越来越严
6. **保持与读者互动**：评论区是金矿
7. **身体第一**：日更对健康消耗很大
8. **不要给读者出选择题**：剧情设计成判断对错题，避免读者分裂弃书
