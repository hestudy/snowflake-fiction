# 番茄小说平台创作指南

> 本指南整合番茄小说网最新规则（2026年2月更新）和实战经验，帮助作品顺利通过推荐评估。
>
> **适用范围**：在番茄小说网发布的长篇网文（10万字+）

---

## 一、推荐验证流程（2026年2月更新）

### 新增「推荐评估」环节

作品达到8万字后，需先通过内容评估才能进入7天推荐验证期。

```
8万字 ──→ 主动发起推荐评估 ──→ 1-3天反馈
                                    │
                    ┌───────────────┴───────────────┐
                    ▼                               ▼
            无需优化                        建议优化
                    │                               │
                    ▼                               ▼
            直接进入7天              可选择：优化后重评 / 直接重评
            推荐验证期                               │
                                                    ▼
                                        最多3次评估后必须进入验证期

15万字 ──→ 平台自动发起评估 ──→ 1-3天反馈
（若仍未进入）                              │
                    ┌───────────────┴───────────────┐
                    ▼                               ▼
            无需优化                        建议优化
                    │                               │
                    ▼                               ▼
            直接进入验证期              7天内需再次发起
                                            │
                                            ▼
                                    超过7天自动进入验证期
```

### 关键节点时间线

| 阶段 | 字数 | 操作 | 反馈时间 | 结果 |
|------|------|------|----------|------|
| 推荐评估 | 8万字起 | 主动发起 | 1-3天 | 评估结果 |
| 自动评估 | 15万字 | 平台发起 | 1-3天 | 评估结果 |
| 推荐验证 | 评估通过后 | 7天小流量 | - | 验证期 |
| 正式推荐 | 满10万字 | 自动进入 | - | 正常推荐 |

### 评估次数限制

- 每本作品最多 **3次**「推荐评估」
- 第3次评估结束后，**必须进入**推荐验证期
- 评估次数 = 作者主动发起 + 平台自动发起

## 二、数据指标标准

### 核心指标及格线

| 指标 | 及格线 | 优秀线 | 说明 |
|------|--------|--------|------|
| **三章追读率** | ≥35% | ≥45% | 100人读第一章，有多少人读到第五章 |
| **完读率(10万字)** | ≥15% | ≥20% | 原创小说标准 |
| **完读率(同人)** | ≥20% | ≥25% | 同人小说要求更高 |
| **追更率** | ≥40% | ≥55% | 40%以上流量倾斜是低追更的5倍 |
| **书架比** | 1:5 | 1:3 | 每1个收藏对应多少人加书架 |

### 黄金三章留存标准

| 检查项 | 标准 | 影响 |
|--------|------|------|
| 第一章出现金手指/生死危机 | 必须 | 留存率高40% |
| 前三章流失率 | <80% | 流失率>80%直接影响推荐 |
| 第一章前300字 | 有爆点 | 决定3秒内是否弃书 |

### 各阶段完读率参考线

| 字数 | 及格线 | 优秀线 |
|------|--------|--------|
| 10万字 | ≥15% | ≥20% |
| 30万字 | ≥15% | ≥18% |
| 50万字 | ≥10% | ≥15% |

## 三、书名与简介优化

### 书名公式

**冲突 + 悬念 + 结果前置**

```
❌ 《我和一只羊》
✅ 《离婚后，前夫跪舔我马甲》

❌ 《霸道总裁爱上我》
✅ 《开局被退婚，我靠直播挖祖坟爆火》

❌ 《冷门古言悬疑》
✅ 《穿成反派亲妈后，我把儿子送进了少管所》
```

### 书名检查清单

- [ ] 是否包含**冲突**（离婚、退婚、穿越）
- [ ] 是否包含**悬念**（马甲、爆火、逆袭）
- [ ] 是否**前置结果**（让读者一眼看到爽点）
- [ ] 是否包含**搜索热词**（方便读者搜索）
- [ ] 长度是否在 **15字以内**

### 简介技巧

**核心原则**：像广告语一样写结果，把最爽的结果放在前两行

**简介三段式结构**：

```markdown
第1段：钩子开场（最吸引人的设定/冲突）
第2段：核心爽点（用"→"展示反转）
第3段：期待感（引导读者入坑）
```

**示例**：

> 她隐藏5年的首富身份曝光。
>
> 他没修炼天赋？→三天筑基，七日金丹。
> 全宗门骂他是废物？→秘境开启当天，他踩着掌门飞剑入场。
>
> 当真相揭开，那些曾经轻视她的人，都将跪在她面前。

### 简介检查清单

- [ ] 是否有 **3个明确钩子**
- [ ] 是否**前置了最爽的结果**
- [ ] 是否用 **"→"** 展示反转效果
- [ ] 是否在前两行抓住眼球
- [ ] 是否与书名形成呼应

## 四、黄金三章详细规范

### 第一章：3秒抓眼球

**核心任务**：从"炸楼"开始写，让读者立刻紧张起来

**开篇公式**：
```
环境异变 + 主角反常行为 + 致命威胁
```

**两种爆款开篇模板**：

| 模板 | 特点 | 示例 |
|------|------|------|
| **危机式开篇** | 主角一出场就陷入绝境 | "穿越即断头台，监斩官喊'午时已到'，她突然想起自己是穿成了被灭门的废柴嫡女" |
| **金手指落地** | 直接跳过铺垫，开局获得能力 | "重生回高考前一天，她脑中响起：'考神系统绑定，押题率100%'" |

**第一章检查清单**：

- [ ] 前300字是否有**爆点**（危机/金手指/反转）
- [ ] 是否避免了**寡淡开场**（"XX是个普通人..."）
- [ ] 是否避免了**信息轰炸**（大量世界观设定）
- [ ] 是否避免了**拖沓入题**（迟迟不进入正题）
- [ ] 主角是否在第一章**登场**

### 第二章：强化冲突

**核心任务**：给主角一个紧迫目标 + 设置具体阻碍

**结构**：
```
目标可视化 + 阻碍具体化 = 读者替主角着急
```

**示例**：

| 类型 | 目标 | 阻碍 |
|------|------|------|
| 重生文 | 三天内找到家族被陷害的证据 | 反派处处监视，连出门都被拦 |
| 系统文 | 1小时内赚到1000块 | 身无分文，还被债主堵门 |

**第二章检查清单**：

- [ ] 主角是否有**紧迫目标**（24小时/3天内）
- [ ] 是否有**具体阻碍**（反派势力/时间限制/能力不足）
- [ ] 世界观是否通过**剧情展开**（不是说明文）
- [ ] 金手指/机遇是否**足够吸引人**

### 第三章：爽点反馈

**核心任务**：让读者尝到甜头，即使只是小胜利

**爽点设计公式**：
```
铺垫（被看不起） → 反转（展示实力） → 反应（围观群众震惊）
```

**第三章结尾钩子类型**：

| 钩子类型 | 示例 | 效果 |
|----------|------|------|
| 悬念钩 | "他不知道的是，真正的危险才刚刚开始……" | 好奇 |
| 转折钩 | "而那个杀手，竟然是他的亲生弟弟。" | 震惊 |
| 期待钩 | "明天，就是他突破筑基期的日子。" | 期待 |
| 危机钩 | "门外传来脚步声——有人来了。" | 紧张 |

**第三章检查清单**：

- [ ] 是否有**小爽点**（打脸小反派/突破小困境/展示隐藏技能）
- [ ] 爽点是否与前两章冲突**呼应**
- [ ] 结尾是否有**钩子**（引导加入书架）
- [ ] 是否让读者感到"**有点爽，想看下一章**"

## 五、关键节点策略

### 8-15万字（推荐评估期）

**发起评估前检查**：

| 检查项 | 标准 | 工具 |
|--------|------|------|
| 前三章钩子 | 足够 | 开篇质量检查 |
| 第5-10章完读率 | ≥60% | 数据预测 |
| 毒点识别 | 无明显毒点 | 角色一致性检查 |
| 书名简介 | 优化完毕 | 书名简介检查 |

**根据数据调整**：

| 数据表现 | 问题定位 | 调整策略 |
|----------|----------|----------|
| 某章跳出率骤增 | 该章内容有问题 | 立即回头修改 |
| 三章追读<35% | 开篇不够吸引 | 优化第一章，加动作/反转 |
| 完读率<15% | 剧情节奏问题 | 检查第5-10章是否有持续钩子 |

### 10万字首秀

**首秀前准备**：

- [ ] 存稿 **15万字**（应对首秀期不断更）
- [ ] 更新时间固定（早8点+晚8点）
- [ ] 每章结尾留钩子

**首秀期策略**：

| 时间 | 操作 | 目的 |
|------|------|------|
| 第1天 | 平台给初始流量 | 关注追读率是否≥15% |
| 第4天 | 检查转化率 | 若<10%准备书测 |
| 每日 | 固定时间更新 | 培养阅读习惯 |

**首秀期警告**：

- ❌ **禁止断更**：直接腰斩流量
- ❌ **禁止大改前三章**：算法判定"内容不稳定"

### 20万字书测

**触发条件**：首秀数据不佳

**书测流程**：

1. 停更2天（调整内容）
2. 上传 **5套书名+封面**
3. 平台自动分发测试
4. 选择追读率最高组合重新发布

**书名组合策略**：

| 方案 | 书名风格 | 封面风格 |
|------|----------|----------|
| A | 原书名 | 原封面 |
| B | 强化"重生"标签 | 暖色调 |
| C | 强化"系统"标签 | 冷色调 |
| D | 强化"打脸"标签 | 冲突感 |
| E | 强化"甜宠"标签 | 甜美感 |

**书测数据标准**：

- 单书名追读率 ≥18% → 可继续推进
- 完读率 ≥50% → 百万字后流量暴增300%

## 六、更新策略

### 各阶段更新要求

| 阶段 | 字数要求 | 更新节奏 | 目的 |
|------|----------|----------|------|
| 验证期 | 日更4000字 | 连续不断更 | 维护验证状态 |
| 首秀期 | 日更6000字 | 固定时间（早8+晚8） | 培养阅读习惯 |
| 日常 | 月更10万字 | 日更4000字 | 拿全勤 |

### 存稿管理

| 场景 | 推荐存稿量 | 用途 |
|------|------------|------|
| 正常运营 | 3-7天（6-14章） | 应对突发情况 |
| 节假日 | 15天（30章） | 预防断更 |
| 首秀前 | 15万字 | 确保首秀期稳定 |

### 更新时间建议

```
推荐更新时间：
├── 早8:00（上班族通勤时间）
├── 中12:00（午休时间）
├── 晚8:00（下班后黄金时段）
└── 晚10:00（睡前阅读时段）
```

### 更新节奏检查

- [ ] 是否固定时间发布
- [ ] 是否有足够存稿应对紧急情况
- [ ] 首秀期是否准备了15万字存稿
- [ ] 更新量是否符合阶段要求

## 八、常见问题与解决方案

### Q1：连续被建议优化怎么办？

**答**：最多3次评估后必须进入验证期

**策略**：
1. 仔细阅读评估反馈
2. 针对性修改问题章节
3. 利用每次反馈提升内容
4. 第3次评估后进入验证期继续优化

### Q2：验证期数据不好怎么办？

**答**：边验证边优化

**策略**：
1. 关注**章节读完率**和**追读率**
2. 定位问题章节（跳出率高的章节）
3. 边更新边修改后续内容
4. 验证期数据不代表最终质量

### Q3：首秀扑街了怎么办？

**答**：20万字启动书测

**策略**：
1. 停更2天调整内容
2. 准备5套书名+封面
3. 平台书测选择最优组合
4. 80%作品能起死回生

### Q4：追更率一直低于35%怎么办？

**答**：检查黄金三章

**策略**：
1. 检查第一章前300字是否有爆点
2. 检查第三章是否有爽点+钩子
3. 检查是否存在"毒点"
4. 考虑重写前三章

### Q5：完读率低于15%怎么办？

**答**：检查第5-10章

**策略**：
1. 第5-10章是算法长期推流的关键
2. 检查是否有持续钩子
3. 检查节奏是否合理
4. 检查是否有逻辑崩坏
