# 综合质量检查 Skill

一键完成文本的全面质量评估，包括：
- **人语化程度检测**（AI痕迹）
- **流水账检测**（内容张力）
- **内容质量评估**（吸引力）

## 使用时机

- 每写完一章后的快速检查
- 发布前的质量确认
- 发现数据不好时的诊断
- 关键章节（开篇、高潮）的精修

---

## 使用方式

### 基础用法

```
/quality-check [文本内容]
```

### 自然语言触发

```
"帮我检查这段内容的质量"
"这个章节质量怎么样"
"综合评估一下这段文字"
"检查一下小说质量"
```

## 检查维度

### 1. 人语化程度检测（权重 30%）

检查 AI 痕迹：

| 检查项 | 说明 |
|--------|------|
| 高频AI词 | "综上所述"、"至关重要"、"深入探讨"等 |
| 句式整齐 | 句子长度过于均匀 |
| 缺乏情感 | 纯信息传递，无情绪描写 |
| 翻译腔 | "在当今社会"、"随着...的发展" |
| 强行升华 | 结尾硬拔主题 |

### 2. 流水账检测（权重 35%）

检查内容张力：

| 检查项 | 说明 |
|--------|------|
| 无冲突 | 事件平铺直叙，没有矛盾 |
| 无情绪 | 缺少角色情感反应 |
| 无期待感 | 读者不关心接下来发生什么 |
| 信息堆砌 | 大量设定/背景灌输 |

### 3. 内容质量评估（权重 35%）

检查吸引力：

| 维度 | 权重 | 检查点 |
|------|------|--------|
| 冲突强度 | 25% | 是否有明确冲突？是否足够强烈？ |
| 情绪密度 | 25% | 角色情绪是否饱满？读者能否共情？ |
| 期待感 | 20% | 读者是否想继续看下去？ |
| 节奏控制 | 15% | 张弛是否得当？ |
| 钩子设计 | 15% | 开头是否抓人？结尾是否有悬念？ |

## 评分标准

### 综合评分等级

| 等级 | 分数 | 描述 | 建议 |
|------|------|------|------|
| 优秀 | 80-100 | 质量过硬，可直接发布 | 保持即可 |
| 良好 | 60-79 | 有小问题，建议优化 | 针对性修改 |
| 及格 | 40-59 | 有明显问题 | 重点改写 |
| 不及格 | 0-39 | 存在严重问题 | 需要重构 |

### 各检查项评分标准

**人语化程度**：
- 90-100：几乎无法分辨是人还是AI
- 70-89：有些AI痕迹，整体自然
- 50-69：明显AI味，需要较多修改
- 0-49：典型AI输出，需要重写

**流水账程度**：
- 80-100：有冲突有情绪有期待
- 60-79：偶有平淡段落
- 40-59：多处缺乏张力
- 0-39：通篇流水账

**内容质量**：
- 80-100：内容扎实，吸引力强
- 60-79：内容合格，有改进空间
- 40-59：有明显问题需优化
- 0-39：存在严重问题

## 与其他 Command 配合

```
推荐工作流：

1. 写完章节后
   /quality-check [正文]

2. 根据报告决定
   ├── 评分≥70 → 进入人语化
   ├── 评分50-69 → 先修复问题
   └── 评分<50 → 重新审视内容

3. 人语化处理
   /humanize-text [正文]

4. 再次检查确认
   /quality-check [处理后的正文]

5. 如果是开篇
   /opening-check [正文]
```

## 注意事项

1. **检查先行**：建议先运行质量检查，再进行人语化
2. **重点排序**：按优先级修改，高优先级问题影响最大
3. **流水账危害**：流水账问题比AI痕迹更影响读者留存
4. **迭代优化**：修改后可再次检查确认效果
